# Plan: QR 인식률 인디케이터

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | QR 코드를 색상·로고·도트 모양 등으로 꾸밀 때 스캔 인식률이 낮아질 수 있으나 사용자가 이를 인지할 방법이 없어 인식 불가능한 QR을 저장·배포하는 위험이 있다. |
| **Solution** | 색상 대비율·데이터 밀도·로고 점유율·도트 모양을 가중 합산한 휴리스틱 인식률 점수를 QR 미리보기 우측에 실시간 표시하고, 저장 시 낮은 인식률이면 경고 다이얼로그를 제공한다. |
| **Functional UX Effect** | 사용자는 미리보기 화면에서 인식률(%)을 즉시 확인하고, 위험 수준일 때 색상 강조로 경고를 인지하며, 갤러리/템플릿 저장 전에 경고를 보고 취소 또는 강행을 선택할 수 있다. |
| **Core Value** | '예쁜 QR'과 '스캔 가능한 QR' 사이의 균형을 사용자 스스로 판단할 수 있는 데이터 제공. 인식률 피드백이 없던 상태에서 신뢰도 있는 디자인 도구로 격상. |

---

## 1. 현황 및 문제점

| 항목 | 현재 | 문제 |
|------|------|------|
| QR 색상 | 어두운 색 필터 없음 | 밝은 색 선택 시 배경과 대비 부족 |
| 로고 삽입 | center 위치 + ECC H 자동 전환 | 사용자가 ECC 레벨 인식 불가 |
| 도트 모양 | 5종 선택 | 하트·별·마름모는 사각보다 인식률 낮음 |
| 데이터 길이 | 무제한 | 긴 URL은 모듈 밀도 증가 → 인식률 저하 |
| 저장 | 경고 없음 | 인식 불가 QR을 그대로 저장 |

---

## 2. 목표

1. **실시간 인식률 점수** 계산 (0~100%) — 휴리스틱 기반
2. **미리보기 우측 배지** 표시 — 숫자(%) + 색상(녹/주/적)
3. **60% 미만 적색 강조** — 아이콘 + 텍스트 강조
4. **저장 경고 다이얼로그** — 80% 미만 시 갤러리 저장·템플릿 저장 진입 전 표시

---

## 3. 인식률 계산 공식

### 3.1 가중치 구성

```
인식률(%) = 대비율 점수(40%) + 데이터 밀도 점수(25%)
          + 로고 점유 점수(20%) + 도트 모양 점수(15%)
```

### 3.2 대비율 점수 (40점 만점)

WCAG 상대 휘도 기반 대비비 계산:

```dart
double contrastRatio(Color fg, Color bg) {
  final l1 = _relativeLuminance(fg);
  final l2 = _relativeLuminance(bg);
  return (max(l1, l2) + 0.05) / (min(l1, l2) + 0.05);
}
```

| 대비비 | 점수(40점 만점) |
|--------|----------------|
| ≥ 7.0  | 40 (최고) |
| ≥ 4.5  | 34 |
| ≥ 3.0  | 26 |
| ≥ 2.0  | 16 |
| < 2.0  | 8 |

- 그라디언트 활성: 최저 대비 지점 추정 (시작색·끝색 중 낮은 쪽 사용)
- 배경 이미지 있음: 대비 추정 불가 → 고정 60% 점수(24점)
- 배경 없음(흰색): QR 색상 vs 흰색(#FFFFFF) 계산

### 3.3 데이터 밀도 점수 (25점 만점)

데이터 문자열 길이 기준:

| 길이 | 점수 |
|------|------|
| ≤ 50자 | 25 |
| 51~100자 | 21 |
| 101~200자 | 16 |
| 201~300자 | 11 |
| > 300자 | 6 |

### 3.4 로고 점유 점수 (20점 만점)

| 조건 | 점수 |
|------|------|
| 로고 없음 | 20 |
| 로고 + center 위치 (ECC H 자동 적용) | 17 |
| 로고 + bottomRight 위치 | 20 |

### 3.5 도트 모양 점수 (15점 만점)

| 도트 모양 | 점수 |
|----------|------|
| square | 15 |
| circle | 14 |
| diamond / star / heart | 11 |

## 4. 기능 요구사항

### 4.1 인식률 배지 (QrPreviewSection 우측)

```
┌─────────────────────────────────────────┐
│                              인식률      │
│   [QR 미리보기 160px]         ████ 85%  │
│                               🟢        │
│                          [돋보기 버튼]   │
└─────────────────────────────────────────┘
```

- 위치: QR 미리보기 Stack 우측 상단 Positioned
- 구성: "인식률" 레이블 + 숫자(%) + 색상 원
- 색상 기준:
  - 80% 이상 → `Colors.green` (녹색)
  - 60~79% → `Colors.orange` (주황)
  - 60% 미만 → `Colors.red` (빨강) + 아이콘 `Icons.warning_amber`

### 4.2 강조 표시 (60% 미만)

```
인식률
⚠️ 58%   ← 빨간 텍스트 + 경고 아이콘
```

- 폰트 굵기 `FontWeight.bold` + 색상 빨강
- 경고 아이콘 `Icons.warning_amber_rounded` (16px)
- 배지 테두리 빨간색으로 강조

### 4.3 저장 경고 다이얼로그 (80% 미만 시)

**갤러리 저장** / **템플릿 저장** 버튼 탭 시:

```
┌────────────────────────────────────┐
│  ⚠️ 인식률이 낮습니다              │
│                                    │
│  현재 인식률: 62%                  │
│  QR 코드가 일부 스캐너에서         │
│  인식되지 않을 수 있습니다.        │
│                                    │
│  주요 원인: 색상 대비 부족         │
│                                    │
│      [취소]    [그래도 저장]       │
└────────────────────────────────────┘
```

- 인식률 수치 + 주요 원인(가장 낮은 점수 항목) 표시
- "취소" → 저장 안 함
- "그래도 저장" → 저장 진행
- 80% 이상이면 경고 없이 바로 저장

---

## 5. 데이터 흐름

```
QrResultState 변경
  → QrReadabilityService.calculate(state, deepLink)
  → ReadabilityScore { total, contrastScore, densityScore, logoScore, dotScore, mainIssue }
  → QrPreviewSection 배지 표시
  → _ActionButtons 저장 시 경고 판단
```

- `QrReadabilityService` — 순수 계산 함수 (no UI)
- `ReadabilityScore` — 불변 값 객체
- 계산은 `qrResultProvider` watch 시마다 재계산 (동기)

## 7. 범위 제외 (Out of Scope)

| 항목 | 이유 |
|------|------|
| 실제 스캔 테스트 (카메라 활용) | 별도 스캔 기능 필요, 복잡도 과다 |
| NFC 태그 저장 경고 | NFC는 별도 플로우 |
| QR 공유 시 경고 | 저장과 달리 임시성이 높음 |

## 8. 성공 기준

| 기준 | 측정 방법 |
|------|-----------|
| 미리보기 우측에 인식률 % 실시간 표시 | 색상 변경 시 즉시 갱신 확인 |
| 60% 미만 시 빨간 경고 아이콘 표시 | 어두운 배경 + 밝은 QR 색 테스트 |
| 갤러리 저장 시 80% 미만 경고 다이얼로그 | 저장 버튼 탭 시 다이얼로그 출현 확인 |
| 템플릿 저장 시 80% 미만 경고 다이얼로그 | 동일 |
| "그래도 저장" 선택 시 정상 저장 | 저장 완료 스낵바 확인 |
