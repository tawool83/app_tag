# Plan: QR 커스터마이징

## Executive Summary

| 관점 | 내용 |
|------|------|
| Problem | QR 코드 하단 라벨이 앱 이름으로 고정되어 있고, 전경색도 검정으로만 출력됨 |
| Solution | QR 결과 화면에 커스터마이징 패널 추가 — 라벨 직접 편집 + 안전 색상 팔레트에서 전경색 선택 |
| UX Effect | 변경 시 QR 미리보기 실시간 갱신, 저장·공유·인쇄 전에 최종 확인 가능 |
| Core Value | 인식 불가 색상 차단(WCAG 대비비 ≥ 4.5:1)으로 안전성 보장, 커스텀 값 이력 저장 |

## 요구사항

### 기능 요구사항
- QR 결과 화면에서 하단 라벨 텍스트 편집 가능 (기본값: 앱 이름)
- 미리 정의된 안전 색상 팔레트에서 QR 전경색 선택 (기본값: 검정)
- 라벨 또는 색상 변경 시 QR 미리보기 실시간 갱신
- 갤러리 저장 / 공유 / 인쇄 시 커스텀된 이미지 반영
- TagHistory에 `qrLabel`, `qrColor` 필드 추가하여 이력 저장

### 색상 안전 기준
- 흰 배경(#FFFFFF) 대비 WCAG 대비비 ≥ 4.5:1 통과 색상만 팔레트에 포함
- 선택 불가 색상 UI 표시 없음 (팔레트 자체에 안전 색상만 노출)

### 비기능 요구사항
- 커스터마이징 패널은 접힌 상태로 시작 (기존 화면 레이아웃 변화 최소화)
- Hive 스키마 하위 호환 유지 (기존 이력 깨지지 않음)
