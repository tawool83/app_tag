# location-map-picker Plan

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 위치 태그 생성 시 위도·경도를 직접 숫자로 입력해야 해 일반 사용자에게 진입 장벽이 높다. |
| **Solution** | 지도 UI에서 탭으로 위치를 선택하면 위도·경도가 자동 입력되도록 변경한다. |
| **Function UX Effect** | 지도를 핀치·스크롤하고 원하는 위치를 탭하면 마커가 찍히고 좌표가 확정된다. |
| **Core Value** | 비전문가도 즉시 사용 가능한 직관적 위치 입력 경험을 제공한다. |

---

## 1. 배경 및 목표

### 1.1 현재 문제
- `LocationTagScreen`은 위도(lat)·경도(lng)를 `TextFormField`로 직접 입력받음
- 일반 사용자는 위도·경도 좌표를 알지 못해 사용 불가
- "지도에서 미리보기" 버튼이 있으나 좌표 확인 용도에 불과함

### 1.2 목표
- 지도 위에서 탭하여 위치를 선택하는 UI로 교체
- 선택된 좌표를 위도·경도 필드에 자동 반영
- 장소명(선택) 입력은 유지

---

## 2. 기능 요구사항

### FR-01: 지도 피커 화면
- `flutter_map` (OpenStreetMap 타일, API 키 불필요) 사용
- 지도 표시 영역: 화면의 약 60% 높이
- 초기 위치: 서울 시청 (37.5665, 126.9780), zoom 12

### FR-02: 위치 선택
- 지도 탭 시 해당 좌표에 마커 표시
- 마커는 핀 아이콘 (`Icons.location_pin`, red)
- 선택된 좌표를 위도·경도 읽기 전용 필드에 표시

### FR-03: 좌표 표시 영역
- 위도·경도 `TextFormField`는 읽기 전용(readOnly: true)으로 유지
- 지도 탭 전에는 "지도에서 위치를 선택하세요" 힌트 표시
- 위도·경도 모두 선택 전 QR/NFC 버튼 비활성화(validator)

### FR-04: 장소명 입력
- 기존 장소명 텍스트 필드 유지 (선택 항목)

### FR-05: 의존성
- `flutter_map: ^7.0.0`
- `latlong2: ^0.9.0`
- 별도 API 키 불필요

## 3. 비기능 요구사항

- 인터넷 연결 없을 때: 지도 타일 로드 실패 시 빈 회색 배경, 마커만 표시
- 플랫폼: iOS, Android 동일 동작

---

## 4. 범위 외 (Out of Scope)

- 역지오코딩(좌표 → 주소 자동 변환) — 별도 API 필요, 이번 범위 제외
- 현재 위치(GPS) 자동 이동 — permission 복잡도, 이번 범위 제외
- 검색 기능 — 이번 범위 제외
